Ниже — готовое ТЗ для Replit на обновление формы “Отправьте свой оффер”, чтобы добавить поля под портфолио/опыт и расширить блок про исполнителя.

⸻

ТЗ: Расширение формы “Отправьте свой оффер”

1) Цель

Повысить конверсию и качество откликов за счёт:
	•	добавления ссылки на портфолио / профили (HH, GitHub и т.д.)
	•	возможности коротко описать релевантный опыт
	•	(опционально) указать стек/навыки
	•	улучшения читабельности и доверия к офферу

2) Контекст

Есть форма отправки оффера с полями:
	•	Ваше имя
	•	Контактная информация
	•	Ваш подход
	•	Общая стоимость
	•	Срок выполнения
	•	Гарантии (необязательно)
	•	Возможные риски (необязательно)

Нужно расширить форму и сохранить текущую логику.

⸻

3) Новые поля и изменения UI

3.1. Новое поле: “Портфолио / профили”

Тип: строка или textarea (предпочтительно textarea в 2–3 строки)
Плейсхолдер: Ссылки на GitHub / HH / LinkedIn / сайт / Notion (можно несколько)
Обязательность: желательно сделать необязательным, но с подсказкой “Рекомендуем добавить для повышения доверия”.

Правила:
	•	Разрешить вставлять несколько ссылок, разделитель: пробел/перенос строки/запятая
	•	Допускается текст вида: HH: ... GitHub: ...
	•	Валидация: если встречается http:// или https:// — проверять корректность URL (минимальная: startsWith http/https + отсутствие пробелов внутри ссылки)
	•	Если пользователь вставил без протокола github.com/... — можно:
	•	либо принять как есть (проще)
	•	либо автодобавлять https:// (если реализуемо быстро)

⸻

3.2. Новое поле: “Опыт и релевантные проекты”

Тип: textarea
Плейсхолдер: Коротко: опыт, похожие задачи, 2–3 примера проектов
Обязательность: необязательно, но рекомендовано.

Ограничения:
	•	0–600 символов (или 0–1000, если тебе удобнее)
	•	Счётчик символов (желательно, но опционально)

⸻

3.3. Новое поле: “Стек / ключевые навыки”

Тип: строка (input)
Плейсхолдер: Напр. Python, FastAPI, PostgreSQL, Docker
Обязательность: необязательно
Ограничения: до 120–200 символов

⸻

3.4. Упорядочивание формы (рекомендуемая компоновка)
	1.	Ваше имя
	2.	Контактная информация
	3.	Портфолио / профили (NEW)
	4.	Опыт и релевантные проекты (NEW)
	5.	Стек / ключевые навыки (NEW)
	6.	Ваш подход
	7.	Общая стоимость
	8.	Срок выполнения
	9.	Гарантии (необязательно)
	10.	Возможные риски (необязательно)
	11.	Кнопка “Отправить оффер”

⸻

4) Логика и валидация

4.1 Обязательные поля (как сейчас)
	•	Ваше имя — required
	•	Контактная информация — required
	•	Ваш подход — required
	•	Общая стоимость — required (формат в п. 4.2)
	•	Срок выполнения — required

4.2 Форматы
	•	Стоимость: разрешить ввод 50000, 50 000, 50 000 ₽, 50000 руб
	•	на бэке (или перед отправкой) нормализовать в число (если есть такая логика)
	•	если пока нет — оставить строкой, но проверить, что не пусто
	•	Срок: 10 дней, 2 недели, до 14 дней — оставить строкой, но не пусто

4.3 Ошибки и подсказки
	•	Ошибки показывать под полем (красный текст)
	•	Для “Портфолио / профили”: если вставлена ссылка с пробелом внутри — подсветить и предложить исправить

⸻

5) Изменения данных/модели

5.1. Frontend: payload отправки оффера

Добавить поля в объект отправки (пример):

{
  "name": "...",
  "contacts": "...",
  "portfolioLinks": "...",
  "experience": "...",
  "skills": "...",
  "approach": "...",
  "price": "...",
  "timeline": "...",
  "guarantees": "...",
  "risks": "..."
}

5.2. Backend/Storage (если есть)
	•	Обновить схему/таблицу/коллекцию: добавить portfolioLinks, experience, skills (nullable)
	•	Убедиться, что старые офферы не ломаются

⸻

6) Отображение оффера (если есть экран просмотра)

Если существует страница просмотра/карточка оффера — добавить:
	•	блок “Портфолио / профили”
	•	блок “Опыт”
	•	блок “Навыки”
Если поле пустое — блок не показывать.

⸻

7) Приёмочные критерии (Acceptance Criteria)
	1.	В форме появились 3 новых поля:
	•	Портфолио / профили
	•	Опыт и релевантные проекты
	•	Стек / ключевые навыки
	2.	Пользователь может отправить оффер, даже если эти поля пустые.
	3.	Если пользователь заполнил “Портфолио / профили” ссылками:
	•	они сохраняются и корректно передаются на бэкенд/в хранилище
	•	(если есть просмотр) отображаются в карточке оффера
	4.	Валидации обязательных полей работают как раньше.
	5.	UI не “прыгает”, всё визуально аккуратно (адаптив не ломается).

⸻

8) Не входит в задачу (Out of scope)
	•	Авторизация/профили пользователей
	•	Прикрепление файлов (PDF, резюме) — можно отдельной задачей
	•	Автоматический парсинг HH/LinkedIn

⸻

9) Дополнительно (по желанию, если останется время)
	•	Чекбокс “Показывать портфолио публично”
	•	Счётчик символов для “Опыт”
	•	Автоформатирование ссылок (добавлять https://)
